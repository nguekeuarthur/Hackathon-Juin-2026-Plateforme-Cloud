# Guide validateur

## 1. Se connecter

Allez sur le portail et cliquez **Se connecter avec Office 365** (compte @git.ch).

## 2. Accéder au tableau de bord du validateur

1. Une fois connecté, vous arrivez sur votre tableau de bord validateur.
2. Vous y voyez un résumé :
   - Demandes en attente : nombre de demandes à traiter
   - VM actives : nombre de machines en cours d'utilisation
   - Coût total estimé : coût cumulé de toutes les VM

## 3. Consulter les demandes en attente

1. Dans votre tableau de bord, cliquez sur **"Demandes en attente"**.
2. Vous voyez la liste de toutes les demandes qui n'ont pas encore été traitées.
3. Pour chaque demande, vous avez :
   - Le nom de l'utilisateur (étudiant ou formateur)
   - La configuration demandée
   - La période souhaitée
   - La date de la demande

## 4. Approuver une demande

1. Dans la liste des demandes en attente, cliquez sur la demande que vous voulez traiter.
2. Consultez les détails.
3. Si tout est conforme, cliquez sur le bouton **"Approuver"**.
4. Confirmez.

L'utilisateur reçoit une notification et peut maintenant accéder à sa VM.

## 5. Refuser une demande

1. Dans la liste des demandes en attente, cliquez sur la demande.
2. Cliquez sur le bouton **"Refuser"**.
3. Obligatoire : indiquez la raison du refus (ex: budget insuffisant, configuration trop élevée, etc.).
4. Confirmez.

L'utilisateur reçoit une notification avec la raison du refus.

## 6. Traiter une demande groupée (formateur)

1. Dans la liste, les demandes groupées sont identifiées par un icône "Groupe".
2. Cliquez dessus pour voir la liste des étudiants concernés.
3. Vous pouvez :
   - Approuver toutes les demandes d'un coup
   - Refuser des demandes individuelles si nécessaire
4. Validez votre choix.

## 7. Surveiller les coûts

1. Dans votre tableau de bord, allez dans **"Tableau de bord des coûts"**.
2. Vous pouvez visualiser :
   - Coût par utilisateur : qui consomme le plus ?
   - Coût par projet : répartition par projet ou classe
   - Évolution dans le temps : tendance des coûts sur la semaine, le mois

## 8. Gérer les prolongations

Les utilisateurs peuvent demander des prolongations :

1. Ces demandes apparaissent dans votre liste de demandes en attente.
2. Elles sont identifiées comme "Prolongation".
3. Vous pouvez les approuver ou les refuser comme une demande classique.

## 9. Historique des actions

1. Allez dans **"Historique"**.
2. Vous voyez toutes les actions que vous avez effectuées :
   - Approbations
   - Refus
   - Modifications
